REGISTRO DE EJECUCIÓN DEL SERVIDOR

1. ¿Qué ocurre cuando ejecutas node server.js?
Node.js inicializa el intérprete de JavaScript, lee el archivo 'server.js', carga el framework Express y abre un proceso continuo en segundo plano que asocia la aplicación al puerto 3000, manteniéndose activo para escuchar peticiones de red.

2. ¿Qué mensaje aparece en la terminal?
Aparece el mensaje de confirmación programado en el callback del método listen:
"Servidor ejecutándose en puerto 3000"

3. ¿Qué sucede cuando visitas la ruta /estado?
El navegador web envía una petición HTTP GET al endpoint. El servidor intercepta esta solicitud y responde enviando un objeto de datos estructurado en formato JSON, el cual se renderiza en la pantalla con el siguiente contenido exacto:
{
  "estado": "Servidor funcionando",
  "servicio": "API comunitaria"
}
